Introducing **Charts**. A collection of chart components that you can copy and paste into your apps.

Charts are designed to look great out of the box. They work well with the other components and are fully customizable to fit your project.

## Composition

We use [solid-charts](https://solidcharts.dev/) under the hood.

We designed the `chart` component with composition in mind. **You build your charts using `solid-charts` primitives and only bring in custom components, such as `ChartTooltip`, when and where you need them**.

```text
ChartContainer
├── Axis
│   ├── AxisLabel
│   └── ChartTooltip
│       └── ChartTooltipContent
└── Bar / Line / Area / other solid-charts primitives
```

```tsx showLineNumbers /ChartContainer/ /ChartTooltipContent/
import { Axis, AxisLabel, Bar } from "solid-charts"

import { ChartContainer, ChartTooltip, ChartTooltipContent } from "~/components/ui/chart"

export function MyChart() {
  return (
    <ChartContainer config={chartConfig} data={data} class="min-h-[200px] w-full">
      <Axis axis="y" position="left">
        <AxisLabel />
      </Axis>
      <Axis axis="x" dataKey="month" position="bottom">
        <AxisLabel />
        <ChartTooltip>
          {({ data }) => <ChartTooltipContent data={data} labelKey="month" />}
        </ChartTooltip>
      </Axis>
      <Bar dataKey="value" fill="var(--color-value)" />
    </ChartContainer>
  )
}
```

We keep the wrapper thin. This means you're not locked into a chart-specific abstraction, and you can keep following the `solid-charts` API as it evolves.

**The components are yours**.

## Chart Config

The chart config is where you define the labels, icons and colors for a chart.

It is intentionally decoupled from chart data.

This allows you to share config and color tokens between charts. It can also work independently for cases where your data or color tokens live remotely or in a different format.

```tsx showLineNumbers /ChartConfig/
import { MonitorIcon } from "lucide-solid"

import { type ChartConfig } from "~/components/ui/chart"

const chartConfig = {
  desktop: {
    label: "Desktop",
    icon: MonitorIcon,
    // A color like 'hsl(220, 98%, 61%)' or 'var(--color-name)'
    color: "#2563eb"
  },
  mobile: {
    label: "Mobile",
    // Or a theme object with 'light' and 'dark' keys
    theme: {
      light: "#60a5fa",
      dark: "#2563eb"
    }
  }
} satisfies ChartConfig
```

## Theming

Charts have built-in support for theming. You can use css variables (recommended) or color values in any color format, such as hex, hsl or oklch.

### Using Colors

To use the theme colors in your chart, reference the colors using the format `var(--color-KEY)`.

#### Components

```tsx
<Bar dataKey="desktop" fill="var(--color-desktop)" />
```

#### Chart Data

```tsx title="components/example-chart.tsx" showLineNumbers
const chartData = [

```

## Tooltip

A chart tooltip contains a label, name, indicator and value. You can use a combination of these to customize your tooltip.

<ComponentPreview name="chart-tooltip" hideCode />

You can turn on or off any of these using the `hideLabel` and `hideIndicator` props and customize the indicator style using the `indicator` prop.

Use `labelKey` and `nameKey` to use a custom key for the tooltip label and name.

Chart comes with the `<ChartTooltip>` and `<ChartTooltipContent>` components. In Solid, render `ChartTooltipContent` inside `ChartTooltip` so you can pass it the active datum.

```tsx title="components/example-chart.tsx"
import { ChartTooltip, ChartTooltipContent } from "~/components/ui/chart"
```

```tsx title="components/example-chart.tsx"
<ChartTooltip>{({ data }) => <ChartTooltipContent data={data} />}</ChartTooltip>
```

### Props

Use the following props to customize the tooltip.

| Prop            | Type                     | Description                                  |
| :-------------- | :----------------------- | :------------------------------------------- |
| `labelKey`      | `string`                 | The config or data key to use for the label. |
| `nameKey`       | `string`                 | The config or data key to use for the name.  |
| `indicator`     | `dot` `line` or `dashed` | The indicator style for the tooltip.         |
| `hideLabel`     | `boolean`                | Whether to hide the label.                   |
| `hideIndicator` | `boolean`                | Whether to hide the indicator.               |

### Colors

Colors are automatically referenced from the chart config.

### Custom

To use a custom key for tooltip labels and names, use the `labelKey` and `nameKey` props.

```tsx showLineNumbers /browser/
const chartData = [
  { browser: "chrome", visitors: 187, fill: "var(--color-chrome)" },
  { browser: "safari", visitors: 200, fill: "var(--color-safari)" }
]

const chartConfig = {
  visitors: {
    label: "Total Visitors"
  },
  chrome: {
    label: "Chrome",
    color: "var(--chart-1)"
  },
  safari: {
    label: "Safari",
    color: "var(--chart-2)"
  }
} satisfies ChartConfig
```

```tsx title="components/example-chart.tsx"
<ChartTooltip>
  {({ data }) => <ChartTooltipContent data={data} labelKey="visitors" nameKey="browser" />}
</ChartTooltip>
```

This will use `Total Visitors` for the tooltip label and `Chrome` and `Safari` for the tooltip names.

## Legend

You can use the custom `<ChartLegend>` and `<ChartLegendContent>` components to add a legend to your chart.

```tsx title="components/example-chart.tsx"
import { ChartLegend, ChartLegendContent } from "~/components/ui/chart"
```

```tsx title="components/example-chart.tsx"
<ChartLegend>
  <ChartLegendContent config={chartConfig} />
</ChartLegend>
```

When `ChartLegendContent` is rendered outside `ChartContainer`, pass `config` explicitly. If you use `nameKey` and need it to infer entries from your dataset, pass `data` too.

### Colors

Colors are automatically referenced from the chart config.

### Custom

To use a custom key for legend names, use the `nameKey` prop.

```tsx showLineNumbers /browser/
const chartData = [

    label: "Safari",
    color: "var(--chart-2)"
  }
} satisfies ChartConfig
```

```tsx title="components/example-chart.tsx"
<ChartLegend>
  <ChartLegendContent config={chartConfig} data={chartData} nameKey="browser" />
</ChartLegend>
```

This will use `Chrome` and `Safari` for the legend names.
